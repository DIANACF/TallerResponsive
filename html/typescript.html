Introducción a Typescript

¿Qué es?

Es una extensión al lenguaje de programación JavaScript que se caracteriza por ampliar su sintaxis en el ámbito de los tipos. Además ,Typescript se conecta de manera más profunda con los editores de código, revisando errores de manera más oportuna.

Sintaxis

Tipos de variables

Variables primitivas

Se refiere a los tipos de datos que son originales de un lenguaje de programación. Para typescript estos son:

  • boolean - Valores verdadero/falso
  • string - Cadenas de texto como "Juan es hijo de Don Luis Fonseca"
  • number - Números enteros y valores de punto flotante
  • null - Un valor nulo o vacío
  • undefined - Se le asigna a una variable cuando no tiene asignado un valor
  • symbol - Usado para asignar un valor único, por lo general se usa como identificador
  • bigint - Representa números enteros muy grandes, más allá del rango que number puede manejar

Variables no primitivas

Son aquellos que se derivan de los tipos primitivos o que son estructuras más complejas.

  • Object - Representa cualquier entidad no primitiva. Los objetos son colecciones de pares clave-valor
  • enum - Una enumeración (enum) es una forma de dar nombres más amigables a un conjunto de valores numéricos

Asignación de tipos de datos

Existen dos formas para la asignación de tipos de datos.Estas son:

La palabra reservada se refiere al uso de let (para variables) o const (para constantes) según sea necesario.

Forma Explicita

En esta forma da a entender el tipo asignado a la variable. Se sigue la siguiente estructura:

palabraReservada nombreDeLaVariable: tipoDeDato

Por ejemplo:

let firstName: string = "Dylan";

Forma Implicita

En esta forma, se tiene que inferir el valor asignado a la variable. Se sigue la siguiente estructura:

palabraReservada nombreDeLaVariable

Por ejemplo:

let firstName = "Dylan";

Clases, métodos, interface y types

Clases

Organizan el código en estructuras reutilizables y permiten definir propiedades y métodos relacionados.

Sintaxis

class NombreDeLaClase{
//atributos
propiedad1: tipo;
propiedad2: tipo;

//constructor
constructor (param1:tipo, param1:tipo,) {
this.propiedad1 = param1;
this.propiedad2 = param2; }

//método
nombreDelMetodo(parametros: tipo): tipoDeRetorno {
// Cuerpo del método }
}

Métodos

Es una función que está definida dentro de una clase y describe un comportamiento que los objetos de esa clase pueden realizar.

Sintaxis


class NombreDeLaClase { nombreDelMetodo(parametros: tipo): tipoDeRetorno {
// Cuerpo del método }
}

Por ejemplo

class Calculadora {
sumar(a: number, b: number): number
{ return a + b;}
}

Interface

Una interface o interfaz es una forma de definir la estructura de un objeto, es decir, qué propiedades y métodos debe tener un objeto, sin preocuparse por su implementación.

Sintaxis

interface NombreDeLaInterfaz {
propiedad1: tipo;
propiedad2: tipo;
metodo1(param1: tipo): tipoDeRetorno;
}

Types

Los types o tipos permiten a los desarrolladores crear alias para combinaciones de tipos existentes o para describir la estructura de datos más complejos.

Usos

1. Alias de tipo

Permiten crear un nombre alternativo para un tipo existente o para una combinación de tipos.

Por ejemplo:

type ID = string | number;

let usuarioId: ID;
usuarioId = "abc123"; // Válido
usuarioId = 123456; // También válido

2. Combinación de tipos

Puedes combinar tipos usando union types (|) o intersection types (&)

Para este ejemplo, se va a utlizar un intersection type (&), por esta razón el objeto debe con todas la propiedades y tipos de los tipos combinados

type TieneNombre = { nombre: string };
type TieneEdad = { edad: number };

type PersonaCompleta = TieneNombre & TieneEdad;

let persona: PersonaCompleta = {
nombre: "Juan",
edad: 25
};

Condicionales

Se utilizan para realizar diferentes acciones en función de diferentes condiciones.

Condicional if

Se utiliza para especificar un bloque de código que se ejecutará, si una condición especificada es verdadera

Sintaxis
if (condición) { //instrucciones que se ejecutarán si la condición es verdadera }

Por ejemplo:

const dia: number = 4;

if(dia >= 1 && dia<=7) { console.log("Es un numero valido para un dia"); }

Condicional else

Se utiliza para especificar un bloque de código que se ejecutará, si la condición propuesta en el if es falsa

Sintaxis
if ( condición ){ //instrucciones si la condición es verdadera}else { //instrucciones si la condición es falsa }

Por ejemplo:

const dia: number = 4;

if(dia >= 1 && dia<=7){ console.log("Es un numero valido para un dia"); }
else{ console.log("No es un numero valido para un dia"); }

Condicional else if

Se utiliza para especificar una nueva condición para probar, si la primera condición es falsa

Sintaxis
if( Condición1) {//instrucciones si la condición es verdadera
} else if( Condición2 ) {//instrucciones si la Condición1 es false y la Condición2 es verdadera
} else { //instrucciones si la Condición1 y Condición2 son falsas }

Por ejemplo:

const dia: number = 4;

if(dia >= 1 && dia<=5){ console.log("El dia es laboral");
}else if(dia>=6 && dia<=7){ console.log("El dia es no laborabl");
}else{ console.log("No es un numero valido para un dia"); }

Condicional switch

Se utiliza para especificar muchos bloques alternativos de código que se ejecutarán

Sintaxis
switch(expresión) {
case x:
// instrucciones
break;
case y:
// instrucciones
break;
default:
// instrucciones
}

Por ejemplo:

const dia: number = 4;

switch(dia){
case 1:
console.log("Lunes");
break;
case 2:
console.log("Martes");
break;
case 3:
console.log("Miercoles");
break;
case 4:
console.log("Jueves");
break;
default:
console.log("No es un numero valido para un dia");
}

Bucles

Se utilizan para iterar sobre colecciones de datos o ejecutar bloques de código repetidamente.

Bucle for

Se utiliza para iterar un número específico de veces.

Sintaxis

for (expresion1; expresion2; expresion3) { //Bloque de código que será ejecutado}
  • La expresión 1 se ejecuta (una vez) antes de la ejecución del bloque de código.
  • La expresión 2 define la condición para ejecutar el bloque de código.
  • La expresión 3 se ejecuta (cada vez) después de que se haya ejecutado el bloque de código.

Bucle for-each

Se utiliza para iterar sobre los elementos de un array y ejecutar una función determinada en cada uno de ellos.

Sintaxis

array.forEach((elemento, indice, arreglo) => {//código a ejecutar});

Por ejemplo

const frutas = ['manzana', 'banana', 'cereza'];

frutas.forEach((fruta) => { console.log(fruta); });

Bucle while

Recorre un bloque de código mientras una condición especificada sea verdadera

Sintaxis

while (condición){ //bloque de código que se ejecutará }

Bucle do-while

Este bucle ejecutará el bloque de código una vez, antes de verificar si la condición es verdadera, y luego repetirá el bucle mientras la condición sea verdadera.

Sintaxis

do { //bloque de código que se ejecutará
} while (condición);

Arrays

En Typescript tienen una sintaxis especifica.

const nombreArray: tipoDeDato[ ] = [ ]

Readonly

La palabra clave readonly evita que se modifiquen los Arrays.

Por ejemplo, en este caso .push no existe en el tipo "readonly" entonces el array no va a cambiar para agregar otro valor

const names: readonly string[] = ["Dylan"];

names.push("Jack");

Inferecia

Se refiere a que Typescript puede inferir el tipo de dato de un Array si este tiene valores.

Por ejemplo, en este caso Typescript infiere que el Array es de tipo number y como no tiene la palabra "readonly" si se pueden agragar valores al Array con .push

const numbers = [1, 2, 3];

numbers.push(4);
