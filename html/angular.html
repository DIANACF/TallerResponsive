Introducción a Angular

¿Qué es Angular?

Angular es un web framework que permite a los desarrolladores crear aplicaciones rápidas y confiables.

Características

Arquitectura Basada en Componentes

Angular organiza las aplicaciones en componentes modulares, cada uno encapsulando lógica utilizando Typescript, HTML, y estilos, lo que facilita el desarrollo, mantenimiento y escalabilidad.

Data Binding Bidireccional

Permite la sincronización automática de los datos entre el modelo y la vista. Esto significa que cualquier cambio en el modelo se refleja instantáneamente en la vista, y viceversa.

Inyección de Dependencias

Angular proporciona un sistema de inyección de dependencias que permite gestionar de manera eficiente los servicios y componentes, promoviendo la modularidad y la reutilización de código.

Directivas

Las directivas son marcadores en el DOM que Angular interpreta y manipula para añadir comportamiento a los elementos HTML. Esto incluye directivas estructurales (*ngIf, *ngFor) y directivas de atributos (ngClass, ngStyle).

Routing

Angular incluye un sistema de enrutamiento que permite definir rutas para navegar entre diferentes vistas o componentes en una SPA (Single Page Application), gestionando el estado de la URL y la navegación de manera fluida.

Formularios Reactivos y Plantilla-Driven

Angular ofrece dos enfoques para manejar formularios: Formularios Reactivos (Reactive Forms) y Formularios Basados en Plantillas (Template-Driven Forms), facilitando la validación y manejo de datos de entrada del usuario.

Animaciones

Angular tiene un módulo de animaciones que permite crear transiciones y animaciones complejas de manera declarativa usando la API de animaciones de Angular.

Angular CLI

Angular CLI es una poderosa herramienta de línea de comandos que facilita la creación, configuración, compilación y despliegue de aplicaciones Angular. Ofrece comandos para generar componentes, servicios, módulos, y más.

Testing integrado

Angular promueve el desarrollo basado en pruebas (TDD) e incluye soporte para pruebas unitarias y de integración mediante herramientas como Jasmine y Karma.

Compativilidad móvil y de escritorio

Desarrollo de aplicaciones móviles y de escritorio.

Configuración

Prerequisitos

  • Node.js - v ^18.9.1 o más reciente
  • Editor de texto: preferiblemente Visual Studio Code
  • Terminal: necesaria para ejecutar los comandos CLI de Angular

Instalación de Angular CLI

Abra una terminal (si está usando Visual Studio Code , puede abrir una terminal integrada ) y ejecute el siguiente comando:

npm install -g @angular/cli

Si se tienen problemas con el comando en Windows o Linux se debe consultar la documentación de la CLI

Creación de un nuevo proyecto

En la terminal, ejecute el comando CLI ng newcon el nombre de proyecto deseado.

ng new < project-name >

Se le presentarán algunas opciones de configuración para su proyecto.Utilice las teclas de flecha e Intro para navegar y seleccionar las opciones que desee.

Si no tiene ninguna preferencia, simplemente presione la tecla Enter para tomar las opciones predeterminadas y continuar con la configuración.

Después de seleccionar las opciones de configuración y la CLI ejecuta la configuración, debería ver el siguiente mensaje:

✔ Packages installed successfully.
Successfully initialized git.

Ejecutar el proyecto localmente

En su terminal, cambie a su nuevo proyecto Angular.

cd el-nombre-de-su-app

Todas tus dependencias deberían estar instaladas en este punto (lo cual puedes verificar verificando si existe una node_modules carpeta en tu proyecto) , por lo que puedes iniciar tu proyecto ejecutando el comando:

npm start

Si todo es exitoso, deberías ver un mensaje de confirmación similar en tu terminal:

Watch mode enabled. Watching for file changes...
NOTE: Raw file sizes do not reflect development server per-request transformations.
➜ Local: http://localhost:4200/
➜ press h + enter to show help

Y ahora puedes visitar la ruta en Local (por ejemplo, http://localhost:4200) para ver tu aplicación.

Signals

Una señal es un contenedor que rodea un valor y notifica a los consumidores interesados ​cuando dicho valor cambia. Las señales pueden contener cualquier valor, desde valores primitivos hasta estructuras de datos complejas.

Lees el valor de una señal llamando a su función getter, lo que permite a Angular rastrear dónde se usa la señal.

Writable signals

Proporcionan una API para actualizar sus valores directamente. Puedes crear señales escribibles llamando a la signalfunción con el valor inicial de la señal:

const count = signal(0);

console.log('The count is: ' + count());

Para cambiar el valor de una Writable signals, puede .set() hacerlo directamente o utilizar .update() para calcular un nuevo valor apartir del anterior:

count.update(value => value + 1);

Las señales escribibles tienen el tipo WritableSignal.

Computed signals

Son señales de solo lectura que derivan su valor de otras señales. Las señales calculadas se definen mediante la computedfunción y especificando una derivación:

const count: WritableSignal< number > = signal(0);

const doubleCount: Signal< number > = computed(() => count() * 2);

La doubleCount señal depende de la count señal. Cada vez que count se actualiza, Angular sabe que doubleCount también es necesario actualizarla.
